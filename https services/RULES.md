# Правила разработки HTTP-модулей 1С УПП

BSL-модули HTTP-сервисов AVGST для 1С УПП. Исходники лежат в репозитории; контракты API дублируются в `lib/docs/seed.ts` портала.

Эталон архитектуры: [pulse/bsl](https://github.com/nikita-yakuschenko/pulse/tree/main/bsl). Образцы кода в этой папке: `ЗаказПокупателя.bsl`, `ПоступлениеТоваровУслуг.bsl`.

---

## 1. Назначение модуля

- Один файл `.bsl` = один HTTP-сервис (сегмент URL) + один метод (обычно `get`).
- Экспортная точка: `Функция GetXxx(Запрос) Экспорт` — **имя функции = имя шаблона URL** в конфигураторе.
- Тонкая обёртка вызывает `GetXxx_Impl(Запрос)`; вся логика — в `_Impl` и вспомогательных функциях.

```bsl
Функция GetReceipts(Запрос) Экспорт
	Возврат GetReceipts_Impl(Запрос);
КонецФункции
```

- В шапке файла (комментарии `//`): объект 1С, список полей ответа, список query-параметров.

---

## 2. URL и именование

| Элемент | Пример |
|---------|--------|
| Полный путь | `{BaseURL}/main/hs/receipts/get` |
| Сегмент сервиса | `receipts` |
| Метод | `get` |
| Функция | `GetReceipts` |

Имя файла = имя объекта метаданных: `ПоступлениеТоваровУслуг.bsl`, `ЗаказПокупателя.bsl`.

---

## 3. Контракт API

### Успех (HTTP 200)

```json
{ "data": [ ... ], "errors": [ ... ] }
```

- `data` — массив найденных объектов (сортировка в запросе: дата убыв., номер убыв., если не оговорено иное).
- `errors` — строки с проблемами парсинга параметров или отдельных документов. **200 + непустой `errors` — норма.**

Заголовки ответа:

- `Content-Type: application/json; charset=utf-8`
- `Access-Control-Allow-Origin: *`

### Ошибка (HTTP 500)

```json
{ "error": "текст из ОписаниеОшибки()" }
```

Только при неперехваченном исключении в `_Impl`.

### Параметр `full`

- Без `full`: базовые поля шапки + **сводки** связанных сущностей (`{ "Есть": ..., "Количество": ... }`).
- С `full=true` (`1`, `true`, `yes`, `да`; ключ в URL — `full` / `Full` / `FULL`): полные ТЧ, вложения (Base64), связанные документы.

### Даты

- Вход: `dd.MM.yyyy`, `dd-MM-yyyy`, `yyyy-MM-dd`.
- Выход: всегда `dd.MM.yyyy` (пустая строка, если дата не заполнена).

### Булево

Истина: `1`, `true`, `yes`, `да` (без учёта регистра).

### Чтение параметров

1. `Запрос.ПараметрыЗапроса.Получить("имя")`
2. fallback: `Запрос.ПараметрыURL["имя"]`

Использовать общую функцию `ПолучитьСтроковойПараметр(Запрос, ИмяПараметра)` — не дублировать однострочные `Попытка ... Исключение КонецПопытки`.

## 4. Структура кода (обязательно)

### 4.1. Разбиение на функции

| Блок | Функция (пример) |
|------|------------------|
| Параметры запроса | `ПолучитьСтроковойПараметр` |
| Булево / даты | `ПреобразоватьВБулево`, `РазобратьДатуСтрокой`, `РазобратьПериодFromTo` |
| Все фильтры | `СобратьФильтры` |
| Пост-фильтры | `ПроходитПостФильтры`, `ПроходитСтроковыйПостФильтр`, `СовпадаетПостФильтр` |
| Запрос к БД | `ПолучитьВыборку...` |
| Одна запись | `Сформировать...ИзВыборки`, `ЗаполнитьШапку...` |
| Ответ HTTP | `СоздатьУспешныйОтвет`, `СоздатьОтветОшибки` |

**Не писать** монолит на 300+ строк в одной функции. Вложенность управляющих конструкций — не глубже 3–4 уровней (требование BSL Language Server).

### 4.2. Общие хелперы (копировать в каждый модуль)

Модули в 1С — отдельные общие модули, поэтому хелперы **дублируются в файле**, но **одинаковы по сигнатуре** между сервисами:

- `ПолучитьСтроковойПараметр`
- `ПреобразоватьВБулево`
- `РазобратьДатуСтрокой`
- `РазобратьПериодFromTo`
- `ПохожеНаЧисло` (для `summ`)
- `ПроходитСтроковыйПостФильтр` / `СовпадаетПостФильтр`

Константы длины даты — через `СтрДлина`, не литералы:

```bsl
КолЧастейДаты = СтрДлина("000");
ДлинаГода = СтрДлина("0000");
ДлинаМесяца = СтрДлина("00");
ДлинаДня = СтрДлина("00");
```

### 4.3. Фильтры

- **В запросе к БД:** период `from`+`to`, `date`, `code` (ПОДОБНО `%...%`), `summ` (точное), `approved` (проведён).
- **Пост-фильтры** (подстрока без учёта регистра): строковые поля представления (`contractor`, `org`, `warehouse`, `responsible` и т.д.).

Ошибки некорректных дат — в `errors`, не HTTP 500.

### 4.4. Сериализация

Перед добавлением в `data` проверять, что объект пишется в JSON (`ЗаписатьJSON` в тестовый `ЗаписьJSON`). При сбое — строка в `errors`, документ не попадает в `data`.

## 5. Стиль BSL (BSL Language Server)

Соблюдать, чтобы панель Problems была чистой.

| Правило | Как соблюдать |
|---------|----------------|
| **InvalidCharacterInFile** | Только ASCII `-` в коде и комментариях. Не использовать типографское тире `—`. |
| **OneStatementPerLine** | Одна инструкция на строку. Запрещено: `Попытка ...; Исключение КонецПопытки;` в одной строке. |
| **MissingCodeTryCatchEx** | В блоке `Исключение` — хотя бы одна исполняемая строка (например `Значение = "";`). |
| **LineLength** | До 120 символов; длинные выражения переносить. |
| **MagicNumber** | Числа 3, 4, 2 для дат — через `СтрДлина("...")` или именованные переменные. |
| **NestedStatements** | Выносить вложенные циклы/условия в отдельные функции. |
| **FunctionShouldHaveReturn** | Мутирующие хелперы без возврата значения - `Процедура`, не `Функция`. |
| **LogicalOrInTheWhereSectionOfQuery** | Не использовать `ИЛИ` в `ГДЕ`; собирать условия динамически (`1 = 1` + `УсловияГде`), как в `ЗаказПокупателя.bsl`. |
| **NestedFunctionInParameters** | Не вызывать функции внутри аргументов `Добавить()`; сначала присвоить в переменную. |
| **DuplicateStringLiteral** | Повторяющиеся заглушки (`"Не указан"`) - в локальную переменную в функции. |
| **MagicNumber (HTTP)** | Коды ответа: `HTTPСтатусУспеха = 200`, `HTTPСтатусВнутренняяОшибкаСервера = 500`. |

### Плохо

```bsl
Попытка ЗначFrom = СокрЛП(Строка(Запрос.ПараметрыЗапроса.Получить("from"))); Исключение КонецПопытки;
// при full — полные ТЧ
```

### Хорошо

```bsl
Попытка
	ЗначFrom = СокрЛП(Строка(Запрос.ПараметрыЗапроса.Получить("from")));
Исключение
	ЗначFrom = "";
КонецПопытки;
// при full - полные ТЧ
```

## 6. Запросы и метаданные

- Текст запроса — многострочный, поля через `|`, с `\n` в параметрах при необходимости.
- Имена полей JSON — **как в документации портала** (кириллица: `Номер`, `ДатаДокумента`, `СуммаДокумента`).
- Пустые ссылочные поля в шапке — заглушки `"Не указан"` / `"Не указана"` / `""` / `0` по образцу существующих модулей.
- Вложения — `Справочник.ХранилищеДополнительнойИнформации`, поле `Объект = &СсылкаДокумента`.

---

## 7. Документация портала

При изменении контракта (новые параметры, поля ответа):

1. Обновить `lib/docs/seed.ts` (страница `methods/*-get`).
2. Выполнить `npm run docs:sync`.

Изменения в `https services/` коммитятся в этот репозиторий вместе с обновлением документации портала.

---

## 8. Чеклист перед переносом в конфигуратор

- [ ] `GetXxx` экспорт, имя = шаблон URL
- [ ] Ответ `{ data, errors }`, CORS, UTF-8
- [ ] `full` обрабатывает три регистра ключа
- [ ] Нет однострочных `Попытка/Исключение`
- [ ] Нет типографских тире в файле
- [ ] BSL LS: 0 problems (или только согласованные исключения)
- [ ] Документация на портале синхронизирована (если менялся контракт)
